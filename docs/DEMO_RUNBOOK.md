# Three-minute demo runbook

Target duration: **2:50–2:55**. Record burned captions and upload platform captions.

| Time | Product moment | Narration purpose |
|---|---|---|
| 0:00–0:15 | Wrong solution receives 100% | Stakes: a student can be graded incorrectly |
| 0:15–0:35 | Start red-team run | Promise: repair the assessment before release |
| 0:35–1:05 | Hypotheses appear; most are rejected | GPT-5.6 proposes, execution decides |
| 1:05–1:25 | Minimal `(1,2,2)` counterexample | Show expected and actual outputs with provenance |
| 1:25–1:45 | Exact patch approval | Instructor remains accountable for the change |
| 1:45–2:05 | Apply, read back, rerun | Close the evidence → action → verification loop |
| 2:05–2:22 | 62.5% → 100%; accepted solutions 100% | Reproducible seeded result, honestly scoped |
| 2:22–2:38 | Frozen result: 53.3% → 93.3%, 0% false kills | Ten-assignment proof, not a one-case claim |
| 2:38–2:46 | Download pytest artifact + audit trail | A real reusable artifact, not a report |
| 2:46–2:52 | Limitation and close | Synthetic v1 ties random-8; real-course review remains |

Test the recording muted and on a phone. Do not describe deterministic fallback as live GPT-5.6,
the synthetic corpus as real-course data, the random-8 tie as search superiority, or a local
artifact write as a live GitHub pull request.
Freeze risky behavior before recording; after freeze, allow only verified P0 fixes and submission
artifacts.
